1. OPART — Operational Art

ยุทธศิลป์ระดับยุทธการ

ความหมายและระดับสงคราม

Operational Art คือการใช้ความคิดสร้างสรรค์และวิจารณญาณของผู้บังคับบัญชาและฝ่ายอำนวยการ เพื่อเชื่อม Strategic → Operational → Tactical ผ่าน Campaign / Major Operation ให้บรรลุ End State

Operational Art = Ends + Ways + Means + Risk
EndsEnd State / Objective
Waysแนวทางใช้กำลังหรือวิธีดำเนินการ
Meansกำลัง ทรัพยากร เวลา เครื่องมือ การสนับสนุน
Riskความเสี่ยงต่อภารกิจ กำลัง เวลา และผลลัพธ์
Strategicทำสงครามเพื่ออะไรNational Objective, War Termination
Operationalจัด Campaign อย่างไรCOG, DP, LOO/LOE, Phase
Tacticalรบอย่างไรBattle, Engagement, Fire, Maneuver

Operational Factors

Operational Factors = Time + Space + Force
Timeเวลา จังหวะ ห้วงปฏิบัติ Tempo, Sequencing, Phasing, Window of Opportunity
Spaceพื้นที่ ระยะทาง ภูมิศาสตร์ ทะเล ช่องแคบ ฐานทัพ SLOC
Forceกำลังรบ ขีดความสามารถ ความพร้อม การจัดกำลัง การสนับสนุน

Campaign — ความหมายในส่วน OPART

Campaign คือชุดของการปฏิบัติการหรือยุทธการหลายส่วนที่สัมพันธ์กัน จัดเรียงตาม เวลา พื้นที่ และกำลัง เพื่อบรรลุวัตถุประสงค์ระดับยุทธศาสตร์หรือระดับยุทธการ และนำไปสู่ End State ที่ต้องการ

Campaign = ชุดการปฏิบัติที่สัมพันธ์กัน → บรรลุ Objective / End State

จำสั้น: Campaign ไม่ใช่ Battle ครั้งเดียว แต่เป็นการจัดหลาย Operation ให้ต่อเนื่องและสอดคล้องกัน

OPART Elements 13 ข้อ

  1. End State
  2. Objectives
  3. Effects
  4. Center of Gravity (COG)
  5. Decisive Points (DP)
  6. Lines of Operation (LOO)
  7. Lines of Effort (LOE)
  8. Direct / Indirect Approach
  9. Operational Reach
  10. Culminating Point
  11. Tempo
  12. Phasing / Transition
  13. Risk

PMESII-PT ไม่ใช่ OPART Element โดยตรง แต่ใช้วิเคราะห์ OE

End / Objective / Effect / Task

End Stateสภาพสุดท้ายที่ต้องการ ไม่ใช่ตัวภารกิจ
Objectiveเป้าหมายที่ต้องบรรลุเพื่อไป End State
Effectผลที่ต้องเกิดจากการกระทำ อาจเป็นกายภาพ พฤติกรรม จิตวิทยา หรือระบบ
Taskกิจที่หน่วยทำ เช่น โจมตี ยึด คุ้มกัน ลาดตระเวน

COG Analysis

COG → CC → CR → CV
COGSource of Power / แหล่งพลังอำนาจหลัก
CCCritical Capability: COG ทำอะไรได้
CRCritical Requirement: ต้องมีอะไรเพื่อให้ CC ทำงาน
CVCritical Vulnerability: จุดอ่อนใน CR ที่โจมตีหรือใช้ประโยชน์ได้

ตัวอย่าง Falklands: COG = Carrier Battle Group; CC = Air/Sea Control; CR = Carrier, Aircraft, Pilot, Fuel, Escort, Logistics, C2; CV = Logistic Ship, Long SLOC, Weather, Aircraft Limited

DP / LOO / LOE / Arrange Operations

DP คือสภาวะ เหตุการณ์ พื้นที่ หน้าที่ หรือขีดความสามารถที่ทำให้ได้เปรียบหรือเข้าใกล้ Objective / COG; DP = Condition ไม่ใช่ Task

LOOLine of Operation มีพื้นที่/ภูมิศาสตร์ชัด เชื่อมการปฏิบัติหรือ DP ตามเวลาและพื้นที่
LOELine of Effort เชื่อมกิจกรรมหลายด้าน ไม่จำเป็นต้องผูกกับพื้นที่ เช่น ข่าวสาร กิจการพลเรือน การทูต เศรษฐกิจ ความชอบธรรม
Directเข้าหา COG โดยตรง
Indirectผ่าน CV / DP เพื่อทำให้ COG เสื่อม
Sequencingเรียงลำดับ DP หรือสภาวะ
Phasingแบ่งขั้นปฏิบัติ
Main Effortความพยายามหลัก เปลี่ยนตาม Phase ได้
Branchแผนเบี่ยงแล้วกลับ LOO เดิม
Sequelไปสู่ Operation หรือ LOO ใหม่
Operational Pauseหยุดสะสมกำลังหรือลดเสี่ยง ไม่ใช่ยุติ

Reach / Tempo / Assessment / Geographic Position

Operational Reachระยะหรือขีดความสามารถในการปฏิบัติต่อเนื่อง ขึ้นกับ Logistics, Fuel, Ammo, Repair
Culminating Pointจุดหมดแรง ทำให้รุกหรือรักษาความได้เปรียบไม่ได้
Tempoจังหวะหรืออัตราการปฏิบัติเทียบกับข้าศึก
Assessmentวัดว่า Effect / Objective เกิดขึ้นจริงหรือไม่
Geographic Positionตำแหน่งทางภูมิศาสตร์ เช่น เกาะ ช่องแคบ ท่าเรือ สนามบิน SLOC
Geostrategic Positionตำแหน่งที่มีคุณค่ายุทธศาสตร์ เช่น ควบคุม SLOC เป็นฐานส่งกำลัง หรือจุดคานอำนาจ

2. OPDESIGN — Operational Design

Design = What/Why, Planning = How

ความหมาย

Operational Design คือกระบวนการคิดเชิงแนวคิด เพื่อทำความเข้าใจปัญหา OE และแนวทางแก้ปัญหา ก่อนนำไปทำแผนรายละเอียด

Design = What / Why | Planning = How

Design Methodology 4 ขั้น

1 Understand Operational Direction → 2 Understand OE → 3 Define Problem → 4 Develop Operational Approach
1 Directionเข้าใจคำสั่ง เจตนารมณ์ Objective, End State, Constraints, Resources, เวลาและทรัพยากรจากหน่วยเหนือ
2 OEเข้าใจ Current State / Desired State / PMESII-PT / Actors / Nodes / Links / Tension และถาม “So What?”
3 Problemระบุปัญหาจริง ไม่ใช่อาการหรือกิจ ชี้ Gap ระหว่าง Current State กับ Desired End State
4 Approachแนวทางเปลี่ยน Current State ไป Desired End State ยังไม่ใช่ COA รายละเอียดหรือ OPORD

PMESII-PT ใช้ทำความเข้าใจ OE

PPoliticalMMilitary
EEconomicSSocial
IInformationIInfrastructure
PPhysical EnvironmentTTime

Problem Statement ที่ดี

  1. Current Condition
  2. Desired End State
  3. Gap / Core Problem
  4. Constraints / Limitations
  5. Implication / Risk if unresolved

หลักจำ: อย่าเขียน Problem เป็นกิจ เช่น “ยกพลขึ้นบก” แต่ต้องเขียนสิ่งที่ขัดขวาง End State

Operational Approach ที่ดี

  • เข้าใจง่าย
  • เชื่อม End State / Objective
  • สอดคล้องกับ OE และข้อจำกัด
  • ใช้ COG, DP, LOO/LOE เหมาะสม
  • นำไปพัฒนา COA ได้
  • ยังเป็นแนวทางกว้าง ไม่ใช่ COA รายละเอียด

Effect / DP / LOO / LOE ใน Design

Effectผลจากการกระทำ อาจเป็นผลกายภาพ พฤติกรรม จิตวิทยา หรือระบบ
DPพื้นที่ เหตุการณ์ เงื่อนไข หรือขีดความสามารถที่ส่งผลต่อ COG / Objective
LOOเชื่อมการปฏิบัติหรือ DP ตามเวลาและพื้นที่
LOEเชื่อมกิจกรรมหลายด้าน ไม่จำเป็นต้องผูกกับพื้นที่ เช่น ข่าวสาร กิจการพลเรือน การทูต เศรษฐกิจ

Design End State / Concept Plan / Detailed Plan / COA Statement

Design End State มาจากคำสั่งและเจตนารมณ์หน่วยเหนือ, Strategic Guidance, Objectives, Desired Conditions, ข้อจำกัด, Current State, Core Problem
Concept Planแผนแนวคิดกว้าง แสดงเจตนารมณ์ วิธีคิด Phase, LOO/LOE, Main Effort
Detailed Planแผนละเอียด ระบุใคร ทำอะไร ที่ไหน เมื่อใด อย่างไร ใช้ปฏิบัติจริง
COA Statement ที่ดีมี Objective, Main/Supporting Effort, Scheme of Maneuver, Phase/Time/Area, Desired Effects, Risk, Success Conditions

3. IPOE — Intelligence Preparation of Operational Environment

รู้เขา / รู้พื้นที่ / รู้ผลกระทบของ OE

Operational Intelligence

Operational Intelligence คือข่าวกรองระดับยุทธการที่สนับสนุน Campaign / Major Operation โดยช่วยให้เข้าใจ OE, Threat, COG, Threat COA, Indicator, Risk และโอกาสในการปฏิบัติ

IPOE ภาพรวม

IPOE = Intelligence Preparation of Operational Environment เพื่อสนับสนุนการวางแผนและการตกลงใจ

Step 1–2Analysis of Operational Environment
Step 3–4Analysis of Threat

IPOE 4 Steps

1 Define OEกำหนด AO / AOI / OE Characteristics
2 Describe OE EffectsPhysical / Non-physical / Time → MCOO
3 Evaluate ThreatCapability / Intent / COG / HVT / HPT
4 Determine Threat COAMLCOA / MDCOA / NAI / Event Matrix

Step 1 Define OE

  • Review Situation
  • Scope Threat: Operational + Environmental Threat
  • Identify OE Characteristics
  • Define AO / AOI หรือ AIR / AII

Products: AO, AOI, OE Characteristics, Overlays, Human Factors, Environmental Graphics

AOArea of Operations: พื้นที่ปฏิบัติการที่รับผิดชอบ
AOIArea of Interest: พื้นที่สนใจที่อาจส่งผลต่อภารกิจ
AIRArea of Intelligence Responsibility: พื้นที่รับผิดชอบด้านข่าวกรอง
AIIArea of Intelligence Interest: พื้นที่สนใจด้านข่าวกรอง

Step 2 Describe OE Effects

ถามว่า “So What?” สิ่งแวดล้อมนี้กระทบการปฏิบัติอย่างไร

  • Physical: Maritime / Land / Air / Space / Weather
  • Non-Physical: Human / Information
  • Time Factors
  • Critical Effects / Issues

Product: MCOO + OE Effect Summary

OAKOC = Observation & Fields of Fire + Avenues of Approach + Key Terrain + Obstacles + Cover & Concealment

MCOO = Modified Combined Obstacle Overlay: Unrestricted, Restricted, Severely Restricted, Key Terrain, Avenues of Approach. MCOO ไม่ใช่แค่แผนที่ แต่คือผลกระทบของ OE ต่อการปฏิบัติ

Step 3 Evaluate Threat

Threat = Capability + Intent + Opportunity
  • Review Threat Situation
  • Capability / Disposition / Intent
  • Strength / Weakness
  • Threat Mission Analysis
  • Threat COG / CF / HVT / HPT
HVTHPT
High Value TargetHigh Payoff Target
ตามกรอบสอบนี้ให้โยงกับ CR: สิ่งสำคัญ/สิ่งจำเป็นที่ข้าศึกต้องใช้เพื่อให้ COG หรือ Critical Capability ทำงานตามกรอบสอบนี้ให้โยงกับ CV: เป้าหมายที่ตีแล้วคุ้ม เพราะกระทบจุดอ่อนสำคัญของข้าศึกและช่วยให้ COA ฝ่ายเราสำเร็จ
สำคัญต่อข้าศึก หากเสียไปข้าศึกลำบากสำคัญต่อฝ่ายเราในการโจมตี/ใช้ผล เพราะให้ผลตอบแทนสูง
มักได้จากการดู Critical Requirementsมักเลือกจาก Critical Vulnerabilities ที่โจมตีได้จริง

จำแบบสอบ: HVT ≈ CR, HPT ≈ CV

คำอธิบาย: HVT คือสิ่งที่มีคุณค่าต่อข้าศึก เพราะเกี่ยวกับสิ่งที่ข้าศึกต้องใช้ ส่วน HPT คือสิ่งที่ฝ่ายเราควรใช้ผล/โจมตี เพราะหากกระทบแล้วช่วยให้แผนฝ่ายเราสำเร็จหรือทำให้ COG ข้าศึกเสื่อม

Step 4 Determine Threat COA

  • Review Threat Mission Analysis
  • Develop Threat COA / Scenario
  • Develop Indicators
  • Draft Collection Plan
MLCOAMost Likely COA = น่าจะทำมากที่สุด
MDCOAMost Dangerous COA = อันตรายที่สุด
NAINamed Area of Interest = พื้นที่เฝ้าตรวจเพื่อพิสูจน์ COA
TAITarget Area of Interest = พื้นที่ที่เหมาะแก่การใช้ผล/โจมตีเป้าหมาย
Event MatrixNAI + Indicator + Time + Meaning + Collection Asset

CCIR / PIR / FFIR / EEFI / TIR และ OODA

CCIRข้อมูลสำคัญที่ ผบ.ต้องใช้ตัดสินใจ
PIRข่าวสารสำคัญเกี่ยวกับข้าศึก/OE
FFIRข่าวสารสำคัญเกี่ยวกับฝ่ายเรา
EEFIข้อมูลฝ่ายเราที่ต้องป้องกันไม่ให้ข้าศึกรู้
TIRความต้องการข่าวกรองระดับยุทธบริเวณ
OODA = Observe → Orient → Decide → Act

ในมุม IPOE: IPOE ช่วยให้ Observe และ Orient ดีขึ้น โดยให้ข้อมูล OE, Threat, Indicator และ Threat COA เพื่อสนับสนุนการตกลงใจ

ดูรายละเอียด OODA ในส่วน Plan Development / Orders & Execution

Plan Development / Orders & Execution / OODA

Plan Developmentเปลี่ยน COA ที่ผู้บังคับบัญชาเลือกแล้วให้เป็นแผนละเอียด พร้อม CONOPS, Task Organization, Support Concept, Sync Matrix และ Annex/Appendix
CONOPSConcept of Operations แนวความคิดในการปฏิบัติของแผนที่เลือก
Task Orgการจัดกำลังเฉพาะกิจให้สอดคล้องกับ COA และ Main Effort
Support Conceptแนวคิดการสนับสนุน เช่น Logistics, Medical, Communication, Sustainment
Sync Matrixตารางประสานเวลา หน่วย กิจ และ Phase เพื่อให้ทุกส่วนปฏิบัติสอดคล้องกัน
OPORDOperation Order คำสั่งยุทธการสำหรับปฏิบัติจริง
FRAGOFragmentary Order คำสั่งเพิ่มเติม/คำสั่งย่อยเพื่อปรับแผนระหว่างปฏิบัติ

OODA Loop ในการปรับแผนและการปฏิบัติ

OODA = Observe → Orient → Decide → Act
Observeตรวจพบและรวบรวมข้อมูลจาก ISR, Sensor, Report, หน่วยปฏิบัติ และสถานการณ์จริง
Orientตีความข้อมูล ทำความเข้าใจ OE, Threat, Indicator, ความเสี่ยง และความหมายต่อภารกิจ
Decideผู้บังคับบัญชาตัดสินใจเลือก/ปรับ COA, ออกคำสั่งเพิ่มเติม, เปลี่ยน Main Effort หรือปรับ Priority
Actปฏิบัติตามแผนหรือคำสั่ง เช่น ออก OPORD/FRAGO, เคลื่อนกำลัง, โจมตี, จัด ISR เพิ่ม หรือปรับ Phase

ความสัมพันธ์กับ Plan Development: แผนที่ดีต้องรองรับการวน OODA ระหว่างปฏิบัติ เพราะสถานการณ์เปลี่ยนตลอดเวลา

ความสัมพันธ์กับ IPOE: IPOE ช่วยให้ Observe/Orient แม่นขึ้น ส่วน ปมส. และ Plan Development ช่วยให้ Decide/Act เป็นระบบ

หลักจำ: ฝ่ายที่ OODA เร็วกว่าและเข้าใจสถานการณ์ดีกว่า จะชิงจังหวะการตัดสินใจและรักษา Tempo ได้ดีกว่า

โครงตอบอัตนัย

IPOE 4 ขั้น

IPOE คือกระบวนการข่าวกรองสนับสนุนการวางแผน มี 4 ขั้น ได้แก่ 1 Define OE กำหนด AO/AOI/OE 2 Describe OE Effects วิเคราะห์ผลกระทบและทำ MCOO 3 Evaluate Threat วิเคราะห์ Capability/Intent/COG/HVT/HPT และ 4 Determine Threat COA จัด MLCOA/MDCOA/NAI/Event Matrix/Collection Plan

ความสัมพันธ์ IPOE Step 1 กับ Step 2

Step 1 ตอบว่า “ดูอะไร/ที่ไหน” โดยกำหนด OE, AO, AOI และลักษณะสำคัญของสภาพแวดล้อม ส่วน Step 2 ตอบว่า “แล้วมีผลอย่างไรต่อ Ops” โดยนำ OE/AO/AOI มาวิเคราะห์ Physical, Non-physical และ Time จนได้ MCOO และ Critical Effects เพื่อนำไปใช้ใน Mission Analysis และ COA

ความเชื่อมโยง IPOE กับ ปมส.

IPOE ให้ข้อมูล OE, Threat และ Threat COA แก่ ปมส. ตั้งแต่ Scoping, Mission Analysis, COA Development, War Game และ Decision โดยเฉพาะ MCOO, Threat COG, HVT/HPT, MLCOA/MDCOA, NAI และ Event Matrix ทำให้การวิเคราะห์ภารกิจและการพัฒนา COA ไม่ใช่การเดา

ความสำคัญของ ปมส.

ปมส. เป็นกระบวนการวิเคราะห์สถานการณ์อย่างเป็นระบบ เพื่อพัฒนา วิเคราะห์ และเปรียบเทียบ COA ลดความคลุมเครือ ลดความเสี่ยง สนับสนุนให้ผู้บังคับบัญชาตกลงใจ และนำไปสู่การออก OPORD ได้อย่างมีเหตุผล
